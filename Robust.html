Robust

Websites must be robust in a way that works reliably on a wide range of user agents, browsers, assistive technologies, mobile devices including elements with old software and hardware. Therefore, users can choose any technology to interact with websites.

Guideline under "Robust" principle

  • Guideline 4.1: Maximize compatibility with current and future user agents, including assistive technologies.

Examples of "Robust" websites

  • Website that are compatible with all browsers.
  • Websites accessible to assistive technologies such as screen reader or Braille Device etc.
  • Websites that include features for alternative inputs such as keyboard navigation, voice commands etc.

Techniques to build robust web sites based on the guidelines

  • Content is compatible with current and future user tools

    This can be achieved with:

    • Responsive design and cross-browser compatibility (the website content or functionality will not change from browser to browser.)
    • Reliable and interpretable mark-up
