Perceivable

Perceivable principle states that information and user interface must be available to the users in a way they can perceive. The majority of users perceive the web content primarily visually, but other might use hearing or touch senses. Nowadays, new emerging technologies may include sensors from smells and taste as well.

Guidelines under "Perceivable" principle

  • Guideline 1.1: Text Alternatives
  • Guideline 1.2: Time-based media
  • Guideline 1.3: Adaptable Content
  • Guideline 1.4: Distinguishable Content

Examples of “Perceivable” websites

  • A website contains a chart diagram that contains vital information. The information contained within the chart diagram must be provided in other forms so that is not only visually perceivable.
  • A website must have sufficient contrast so that the background can be distinguished from the text by all users including colour-blind and low-vision users.

Techniques to build perceivable web sites based on the guidelines

  1. Provide text alternatives for non-text content. (Guideline 1.1)

    • Iincludes equivalents for images, buttons, icons, charts, video and audio
    • In case of an image the “alt” tag must provide a text alternative equivalent to that image. If the image has no valuable meaning the “alt” tag must remain empty. The words within the alt text must be concise and on point.
    • Useful for users with visual impairments and use assistive technologies such as screen reader and improves search engine optimization. Also useful with slow internet connection.

    EXAMPLE: Providing alt text for image

    < body >
      <img src = example_pict.jpg alt = "sunny intervals" >
    < /body >


  2. Provide captions and other alternatives for multimedia. (Guideline 1.2)

    • Time-based media such as video or audio, must be supported in other forms so that users can convey the information in multiple ways.
    • Captions, transcripts, sign language are other forms that users can convey the information of time-based media.

    EXAMPLE: Default captions

    Some online applications automatically inlcude captions into movies to ensure that users that do not know how to turn on captions will be able to view the movie

  3. Present content in different ways (Guideline 1.3)

    • Need to consider what content will look like if we remove the styling.
    • Headings, lists, tables, input fields and content structures must be marked-up so that users can convey the same information as when styling is applied.
    • Content must be compatible with assistive technologies

    EXAMPLE: Include a skip navigation link at the top of the document so that the users can skip over the navigation links that are placed into the header when CSS is not available.

    Illustration: HTML File

    < a href = “#mainContent” > Skip to main content </a>

    In the CSS file we can visually hide this link but still make it available for keyboards and CSS-less users:

    .skipNavigation{
    Boarder:0;
    clip:rect(1px,1px,1px,1px);
    overflow:hidden;
    padding:0;
    position:absolute;
    heigh:1px;
    width:1px;
    }


  4. Make content easier to see and hear (Guideline 1.4)

    • We need to make content distinguishable
    • Some Good practises to make distinguishable content are:
      1. Don’t use colour as the only way for conveying information
      2. High contrast level between foreground and background.
      3. Allow users to resize text without loss of content or functionality
