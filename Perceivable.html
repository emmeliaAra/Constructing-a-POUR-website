Perceivable

Perceivable principle states that information and user interface must be available to the users in a way they can perceive. The majority of users perceive the web content primarily visually, but other might use hearing or touch senses. Nowadays, new emerging technologies may include sensors from smells and taste as well.

Guidelines under "Perceivable"

  • Guideline 1.1: Provide text alternatives for any non-text content so that it can be changed into other forms people need, such as large print, braille, speech, symbols or simpler language.
  • Guideline 1.2: Provide alternatives for time-based media.
  • Guideline 1.3: Create content that can be presented in different ways (for example simpler layout) without losing information or structure.
  • Guideline 1.4: Make it easier for users to see and hear content including separating foreground from background.

Examples of “Perceivable” principle

  • A website contains a chart diagram that contains vital information. The information contained within the chart diagram must be provided in other forms so that is not only visually perceivable.
  • A website must have sufficient contrast so that the background can be distinguished from the text by all users including colour-blind and low-vision users.

Techniques to build perceivable web sites

  1. Provide text alternatives for non-text content.
    • Iincludes equivalents for images, buttons, icons, charts, video and audio
    • In case of an image the “alt” tag must provide a text alternative equivalent to that image. If the image has no valuable meaning the “alt” tag must remain empty. The words within the alt text must be concise and on point.
    • Useful for users with visual impairments and use assistive technologies such as screen reader and improves search engine optimization. Also useful with slow internet connection.
