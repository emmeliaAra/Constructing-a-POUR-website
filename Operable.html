Operable

Operable principle states that the user interface and navigation components must be operable by any user despite any disability, hardware, software or technologies that they use. Users can successfully use controls, buttons, navigation and other interactive elements to access the website.

Guidelines under "Operable" principle

  • Guideline 2.1: Keyboard Accessible
  • Guideline 2.2: Provide users enough time to read and use content.
  • Guideline 2.3: Do not design content in a way that is known to cause seizures.
  • Guideline 2.4: Provide ways to help users navigate, find content, and determine where they are.

Examples of “Operable” websites

  • A navigation menu with a submenu section revealed when a user hovers over it with a mouse must have the same functionality when it receives focus either with the keyboard or any other interactive device.
  • An online form requires multiple sections from a drop-down menu but only applicable by pressing Ctrl + selecting elements from menu. This is not operable as not all users can do these two steps simultaneously.

Techniques to build operable web sites based on the guidelines

  1. Enable keyboard functionality

    Users must be able to access all functionality that is available by mouse by using a keyboard.Since no pointer is used to indicate focus, ensure that focus is indicated appropriately.

  2. Give users time to read and use the content

    Not everybody reads and types with the same speed. Therefore, allow users to adjust the time limits based on their needs, re-authenticate if the session expires and pause, stop or hide moving/blinking/scrolling content.

  3. Avoid content that can cause seizures.

    Flashes at certain patterns or rates can cause users to have seizures.If possible, flashing content and animations must be avoided otherwise the user must be informed before the content is presented.Alternatives must be provided to perceive the information contained within flashing content or animations.

  4. Make navigation easier

    Clear and well-organized website allow users to navigate effectively. Techniques in this area of site operability include:

    • Clear page titles
    • Include frequent and clear headers to structure the page
    • Provide descriptive text for links as opposed to “Read More”
